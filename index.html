نظرة عامة على ai-camera-backend-fresh

هذا المشروع هو مركز النظام. منه تدار الحسابات، الكاميرات، الحضور، المخالفات، وإرسال البيانات للتطبيقات والعمال (Recorder وInference Worker).

  • Core: Laravel 12 + PHP 8.4
  • Web UI: Inertia + Vue 3 + Vite
  • Security: Sanctum + Fortify + Roles/Permissions
  • Multi-tenant: Stancl Tenancy

شرح مبسط لغير التقنيين

إذا كنت مديرًا أو مشرفًا: اعتبر هذا المشروع "غرفة التحكم" الرئيسية للنظام. كل ما يحدث في التطبيق يمر من هنا: تسجيل دخول الموظف، تسجيل الحضور، الإشعارات، المخالفات، وتقارير الإدارة.

من يستخدمه؟

  • السوبر أدمن: يدير الشركات/المستأجرين والباقات والإعدادات العامة.
  • أدمن الشركة: يدير الموظفين والمواقع والكاميرات والحضور والمخالفات.
  • التطبيق (الموظف): يرسل طلباته ويستقبل البيانات من API الموظف.

ماذا يحدث يوميًا؟

  1. الموظف يسجل الدخول من التطبيق.
  2. يرسل Check-in/Check-out للحضور.
  3. النظام يسجل الأحداث ويرسل الإشعارات.
  4. المشرف يراجع المخالفات والتقارير من لوحة الويب.

المعمارية التقنية

Mobile App (Expo)
    -> Tenant Domain /api/employee/*
    -> Tenant Database

Recorder Worker
    -> Central Domain /api/recorder/*
    -> segment lifecycle + storage config

Inference Worker
    -> Central Domain /api/inference/*
    -> claim jobs + ingest + face profiles

Tenant Dashboard
    -> /dashboard/*

Super Admin Dashboard
    -> /super/*

مهم جدًا: تطبيق الموبايل يتصل بدومين Tenant وليس الدومين المركزي.

الإعداد والتشغيل المحلي

نفّذ الخطوات التالية داخل ai-camera-backend-fresh:

cd ai-camera-backend-fresh
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
npm install
npm run dev
php artisan serve

ملاحظات التنفيذ سطرًا بسطر

  1. cd ai-camera-backend-fresh

    ينقلك إلى مجلد المشروع الصحيح قبل أي أمر آخر.

  2. composer install

    يثبّت حزم Laravel وPHP داخل vendor.

  3. cp .env.example .env

    ينسخ ملف البيئة. في PowerShell استخدم Copy-Item .env.example .env.

  4. php artisan key:generate

    ينشئ APP_KEY المطلوب لتشفير الجلسات والبيانات.

  5. php artisan migrate --seed

    ينشئ الجداول ويضيف بيانات البداية (seeders).

  6. npm install

    يثبّت مكتبات الواجهة (Vite/Vue/JS).

  7. npm run dev

    يشغّل Vite لتجميع الواجهة أثناء التطوير.

  8. php artisan serve

    يشغّل خادم Laravel المحلي للوصول إلى النظام من المتصفح.

للتطوير اليومي السريع: composer dev

Seeders وبيانات البداية

Seeders هي ملفات تملأ قاعدة البيانات ببيانات أولية (مثل الأدوار، المواقع، مستخدمين تجريبيين) حتى يعمل النظام مباشرة بعد التثبيت.

الأوامر الأساسية

php artisan db:seed
php artisan db:seed --class=CentralDatabaseSeeder
php artisan db:seed --class=TenantDatabaseSeeder
php artisan tenants:seed

ملاحظات أوامر Seeders سطرًا بسطر

  1. php artisan db:seed

    يشغّل seeder الافتراضي DatabaseSeeder.

  2. php artisan db:seed --class=CentralDatabaseSeeder

    يملأ قاعدة البيانات المركزية (Central) فقط.

  3. php artisan db:seed --class=TenantDatabaseSeeder

    يملأ بيانات المستأجر الحالي إذا كنت داخل سياق tenant.

  4. php artisan tenants:seed

    يشغّل seeder المستأجرين على جميع الـ tenants أو المحدد منهم بالأمر.

المصادقة والصلاحيات

  • لوحة الويب تعتمد جلسات web guard.
  • مسارات /super/* تتطلب super-admin.
  • API الموظف يستخدم Bearer Token من Sanctum (موديل Employee).
  • API recorder/inference يستخدم RecorderClient وInferenceWorkerClient.

Checklist التكامل مع التطبيق

  1. اضبط EXPO_PUBLIC_API_BASE_URL على دومين Tenant الصحيح.
  2. اختبر /employee/auth/login وتأكد أن access_token يعود بشكل صحيح.
  3. اختبر /employee/attendance/today + /employee/violations.

استكشاف الأعطال الشائعة

خطأ 401 من التطبيق

غالبًا التوكن منتهي/غير صالح أو الاتصال بدومين غير صحيح.

خطأ route للمخالفات

يعني endpoint غير متوفر في البيئة الحالية أو الدومين ليس Tenant domain الصحيح.

مشاكل check-in/check-out

راجع الحقول: action, latitude, longitude وحدود rate limit.

شرح مبسط لغير التقنيين

هذا هو تطبيق الموظف اليومي. الموظف يفتح التطبيق، يسجل دخول، ثم يسجل حضوره، ويتابع الإشعارات والمخالفات إن وجدت.

ما الذي يراه الموظف؟

  • الرئيسية: حالة الحضور الحالية وزر Check In / Check Out.
  • السجل: سجل الحضور الشهري.
  • المخالفات: المخالفات وتفاصيلها وإرسال اعتراض.
  • التنبيهات: الإشعارات الجديدة.
  • الملف الشخصي: بياناته الأساسية وإحصاءات مختصرة.

  • useLogoutMutation يمسح الجلسة والكاش.

تدفق الحضور
